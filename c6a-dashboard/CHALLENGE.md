# Challenge C6A：提交数据仪表盘（Submission Dashboard Builder）

> 🔥 从评审者到系统构建者——不只是看别人的作业，而是构建一个**管理所有人作业的系统**。
> 本质：用 C4A 评审技能收集全部提交数据，然后用 Agent（OpenClaw 或同等工具）构建一个本地网站，把所有人的提交变成**可浏览、可筛选、可追踪**的仪表盘。

---

## 🤔 这是什么

Elite 20 课程产生了大量提交物——每个同学、每个挑战都有多个文件。但目前这些提交散落在群文件、聊天记录、各种文件夹里，**没有人能一眼看清全貌**。

C6A 要求你：

1. **用 C4A 技能**收集和整理所有同学的提交数据
2. **用 Coding Agent**（OpenClaw / Claude Code / Cursor / 其他）构建一个本地网站
3. 网站要清晰展示：**谁提交了什么、什么时候提交的、每个挑战的完成情况**
4. 数据也要导出为 **Excel 表格**供离线查看

> 💡 这是**系统集成 + 数据工程 + 前端展示**的综合挑战。你不是在做一个小工具——你是在构建一个**课程管理系统的原型**。

## 🎯 四级任务

### Level 1：数据收集与结构化（Bronze 🥉）

> **把散落的提交数据整理成结构化格式。**

- 用 C4A 评审技能梳理所有同学的提交情况
- 为每个提交记录建立结构化条目，包含：
  - **学生姓名**
  - **挑战编号**（C2A / C4B / C4C / C4D / C5A …）
  - **提交时间**
  - **提交文件清单**（哪些文件交了）
  - **完成状态**（✅ 完整 / ⚠️ 部分 / ❌ 缺失）
  - **必填项检查**（AI日志是否提交？关键产出是否到位？）
- 输出为 **JSON** 或 **CSV** 文件

**交付：** 结构化数据文件（JSON/CSV）+ 数据收集说明文档

---

### Level 2：Excel 报表（Silver 🥈）

> **把数据变成可读的 Excel 表格。**

在 Level 1 基础上，生成一份 Excel 工作簿：

| Sheet | 内容 |
|-------|------|
| **总览** | 学生 × 挑战 矩阵（✅ / ⚠️ / ❌）+ 完成率统计 |
| **按学生** | 每个学生的所有提交详情 |
| **按挑战** | 每个挑战的所有学生提交情况 |
| **时间线** | 按提交时间排序的全部提交记录 |
| **缺失项** | 标红所有缺少必填项（AI日志等）的条目 |

用 Agent 自动生成，不要手动填表。

**交付：** Excel 文件 + 生成脚本

---

### Level 3：本地网站仪表盘（Gold 🥇）

> **用 Coding Agent 构建一个交互式网站。**

用 **OpenClaw、Claude Code、Cursor** 或其他 Coding Agent 构建本地网站，功能包括：

| 功能 | 说明 |
|------|------|
| **总览页** | 全班完成率可视化（进度条 / 热力图 / 饼图） |
| **学生详情页** | 点击学生名字查看所有提交 |
| **挑战详情页** | 点击挑战编号查看所有学生的完成情况 |
| **筛选/排序** | 按学生、挑战、时间、状态筛选 |
| **文件链接** | 点击文件名可打开/下载对应提交物 |
| **缺失提醒** | 高亮显示缺少必填项的条目 |

推荐技术栈：

| 方案 | 说明 |
|------|------|
| **HTML + Tailwind + JS** | 纯静态，零依赖，浏览器直接打开 |
| **React / Vue** | 更复杂的交互 |
| **Flask / FastAPI + Jinja** | Python 后端渲染 |
| **Streamlit** | 最快的 Python 数据仪表盘 |

**交付：** 可运行的网站代码 + 截图/录屏 + 部署说明

---

### Level 4：自动化 + 实时更新（Platinum 💎）

> **不只是一次性报表——构建一个可持续更新的系统。**

在 Level 3 基础上：

- **自动扫描：** 脚本自动扫描提交文件夹，发现新提交自动更新数据
- **校验引擎：** 自动检查每个提交是否符合命名规范、是否包含必填文件
- **评审集成：** 嵌入 C4A 评审标准，自动对提交物做初步质量评估
- **通知系统：** 生成每日/每周提交状态摘要（可导出为 Markdown 发群）
- **多课程扩展：** 架构支持添加新挑战、新学生，不需要改代码

**交付：** 完整系统 + 自动化脚本 + 技术文档 + GitHub repo

## ⚡ 起步指南

### Step 1：收集数据（Day 1–2）

用 C4A 评审技能（或手动）梳理群文件中的所有提交：

```
对你的 Coding Agent 说：
"帮我扫描这个文件夹，找出所有符合 Elite 20 命名规范的提交文件，
按学生和挑战分类，输出为 JSON。"
```

### Step 2：定义挑战要求（Day 2）

为每个挑战（C2A、C4B、C4C、C4D、C5A …）定义必填文件清单，这样系统才能自动判断完成状态。

### Step 3：生成 Excel 报表（Day 3）

```
对 Agent 说：
"用这份 JSON 数据生成一个 Excel 工作簿，包含总览、按学生、按挑战、
时间线四个 Sheet，用条件格式标注缺失项。"
```

### Step 4：构建网站（Day 3–6）

```
对 Agent 说：
"用这份 JSON 数据构建一个本地 HTML 网站，包含总览仪表盘、
学生详情页、挑战详情页，支持筛选和排序。用 Tailwind CSS 做样式。"
```

### Step 5：测试 + 提交（Day 7）

让一个同学试用你的网站，确认数据准确、功能可用。

## 评审标准

| 维度 | 权重 | 说明 |
|------|------|------|
| **数据完整性** | 25% | 收集了多少学生、多少挑战的数据？遗漏了吗？ |
| **展示质量** | 25% | 网站/Excel 是否清晰、可读、易用？ |
| **自动化程度** | 20% | 数据收集和校验是自动的还是手动的？ |
| **完成级别** | 15% | Level 1–4 到哪一级？ |
| **可复用性** | 15% | 下一期课程能直接用你的系统吗？ |
