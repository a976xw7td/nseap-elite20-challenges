# Challenge 7：内容传播（Content Distribution）

> **目标：构建个人影响力。**
> 你做的东西再好，没人知道就等于不存在。C7 训练的是让世界看到你的能力。

---

## 一、挑战目标

选择一个内容平台（微信公众号 或 知乎），**持续发布内容**，在群内分享，用数据证明影响力。

C7 是 Elite20 成长路径的终点——"影响能力"：

```
C1-C2 你学会了东西
  → C3 你能思考和讨论
    → C4 你把能力变成可复用的技能
      → C5 你把它开源了
        → C6 你把它变成产品了
          → C7 你让世界知道了
```

从 C4 到 C7 是一条完整的价值放大链。C4 被 10 人用是 10 倍。C7 被 1000 人读是 1000 倍。**你在 C1–C6 做的每一件事，都是 C7 的素材。**

---

## 二、为什么是内容传播

| 能力 | 内容传播怎么训练 |
|------|-----------------|
| **表达能力** | 把复杂的事情讲到外行也能懂 |
| **受众思维** | 不是"我想说什么"，而是"读者需要什么" |
| **持续输出** | 一篇文章是运气，持续输出才是能力 |
| **数据驱动** | 阅读量、点赞、转发不会撒谎——市场是最诚实的老师 |
| **个人品牌** | 在AI时代，有公开作品集的人比只有简历的人有 10 倍优势 |

写给别人看，和写给自己看，是完全不同的能力。C7 训练的就是前者。

选一个主平台，深耕。不要分散精力。

### 方案 A：微信公众号

| 维度 | 说明 |

| 发布节奏 | 建议每周 1 篇，至少提交 2 篇 |

### 方案 B：知乎

| 维度 | 说明 |
|------|------|
| 适合内容 | 回答问题、技术科普、方法论、经验总结 |

## 四、内容从哪来

**你不需要凭空创造内容。** C1–C6 的每一步都是 C7 的素材矿：

| 来源 | 可以写什么 | 示例标题 |
|------|-----------|----------|
| **C1 翻译** | 翻译经验、AI 翻译工具对比、学术资料获取方法 | 「我用 Claude 3 小时翻译了一门斯坦福课程——完整方法公开」 |
| **C2 论文** | 写论文的 AI workflow、LaTeX 入门、学术写作方法 | 「零基础用 AI 写出可投稿论文：我的 AI4Math 写作全过程」 |
| **C3 讨论** | 群内讨论精华整理、学习反思、认知变化 | 「在 Elite20 群里待了一个月，这 5 个认知彻底改变了我」 |
| **C4 技能** | 技能教程、使用指南、踩坑记录、用户反馈 | 「我做了一个 AI 技能被 20 个人用了——从 0 到 1 的完整复盘」 |
| **C5 GitHub** | 开源项目介绍、技术实现拆解、从零建 repo | 「从来没用过 GitHub？这篇文章教你 30 分钟发布第一个项目」 |
| **C6 Web App** | 产品设计思路、AI 集成实践、部署踩坑、用户反馈 | 「不会写代码的我，用 AI 做了一个 Web App 上线了」 |
| **学习过程** | AI 使用心得、Vibe Coding 体验、工具对比 | 「Vibe Coding 到底是什么？我用一周时间搞明白了」 |
| **拿来过程** | 你怎么找到好资源、怎么评估、怎么改进 | 「别从零开始：我如何用"拿来主义"3 天做完别人 3 周的活」 |

**核心思路：把你的过程写出来，而不是只写结果。** 读者不关心你做了一个翻译，他们关心你**怎么做的**、踩了**什么坑**、用了**什么工具**。过程比结果有 10 倍的内容价值。

## 五、爆款内容的结构

不需要追求"爆款"，但好的结构能让同样的内容多 3-5 倍阅读量。

### 5.1 标题公式

| 公式 | 示例 |
|------|------|
| 数字 + 结果 | 「3 个 AI 工具让我的翻译效率提升 10 倍」 |
| 反常识 | 「不会编程的人，反而更适合做 AI 产品」 |
| 过程 + 结果 | 「我用 Claude 写了一篇论文，从零到可投稿只花了 2 天」 |
| 痛点 + 方案 | 「每次做 PPT 要花 3 小时？试试这个 AI 技能」 |
| 第一人称 + 真实 | 「作为 AI 小白，我在 Elite20 的第一周」 |

**避免：** 太学术的标题（「基于 KSTAR 认知架构的技能编译方法研究」→ 没人点）。

### 5.2 正文结构（推荐）

```
1. 开头：痛点/故事/反常识（3 句话抓住读者）
2. 背景：为什么这件事重要（简短）
3. 过程：你具体做了什么（核心部分，配图/代码/截图）
4. 结果：做出来什么效果
5. 反思：踩了什么坑、学到什么
6. 行动：读者看完可以做什么（附资源链接）
```

**关键原则：**
- 每段不超过 3 行（手机阅读体验）
- 每 300 字配一张图或一个代码块
- 重要结论加粗
- 结尾留一个互动问题（提升留言率）

### 5.3 排版要求

公众号排版直接影响阅读体验。推荐：

- 字号：正文 15-16px，标题 18-20px
- 行间距：1.75 倍
- 段间距：15-20px
- 配色：不超过 3 种颜色
- 图片：宽度统一，加圆角或阴影

**可以用 AI 生成排版：** 写完内容后，让 AI 转成公众号兼容的 HTML 格式（参考 `wechat-html-converter` 技能）。

## 六、AI-First 在 C7 的应用

内容创作是 AI 最擅长的领域之一。**全流程 AI 驱动：**

| 阶段 | AI 怎么用 | 具体操作 |
|------|----------|----------|
| **选题** | 让 AI 分析热门话题、生成选题列表 | "根据 Elite20 的 7 个挑战，帮我生成 10 个公众号选题" |
| **大纲** | 让 AI 生成文章结构 | "这篇文章的目标读者是大学生，帮我列一个大纲" |
| **初稿** | 让 AI 写完整初稿 | "按这个大纲写一篇 2000 字的公众号文章" |
| **标题** | 让 AI 生成 10 个候选标题 | "给这篇文章起 10 个标题，要求吸引点击" |
| **改稿** | 让 AI 优化可读性 | "这篇文章太学术了，改成口语化风格，适合非技术读者" |
| **排版** | 让 AI 生成公众号 HTML | "把这篇 Markdown 转成微信公众号兼容的 HTML" |
| **数据分析** | 发布后让 AI 分析数据 | "这篇阅读量 500，上篇 200，帮我分析差异原因" |

**你的角色：** 方向决策、素材提供（你的真实经历）、质量把关、发布运营。AI 写的初稿 + 你的真实经历 = 既有质量又有温度。

**段位参考：**

| 段位 | 在 C7 的表现 |
|------|-------------|
| 🟢 入门 | 让 AI 写一篇，简单改改就发 |
| 🔵 进阶 | 设计大纲，AI 分段生成，逐段审校优化 |
| 🟣 驾驭 | 建立完整的内容生产 workflow：选题→大纲→初稿→改稿→排版→发布→分析→下一篇 |
| 🔴 创造 | 把内容生产 workflow 变成一个 Skill（C4 联动），自己用也教别人用 |

## 七、拿来主义在 C7 的应用

内容创作的拿来主义 ≠ 抄袭。而是：

| 拿来什么 | 从哪拿 | 怎么改 |
|----------|--------|--------|
| 文章结构 | 爆款文章的行文节奏 | 替换成你自己的内容和案例 |
| 标题技巧 | 10W+ 文章的标题公式 | 套用公式，填入你的主题 |
| 排版模板 | 公众号排版工具 / 其他优秀公众号 | 调整配色和字号适配你的风格 |
| 选题方向 | 同领域热门文章 / 群内讨论热点 | 加入你自己的视角和实操经验 |
| 观点框架 | 别人的方法论 / 理论 | 用你的实践去验证或反驳 |

**ATTRIBUTION 要求：** 在文章末尾或 AI 日志中写明参考了哪些文章/结构/方法。

**最有价值的拿来：** 把群内讨论（C3）整理成公众号文章。讨论是半成品的内容，你加上结构和背景就是一篇完整文章。写的时候 @ 原始讨论者——对你是 C7 贡献，对他们是被引用的认可。

## 八、评判标准

| 权重 | 指标 | 怎么衡量 |
|------|------|----------|
| 🥇 高 | **阅读量 / 互动量** | 阅读数、点赞、在看、收藏、评论、转发 |
| 🥇 高 | **内容质量** | 有没有真实经验？有没有可操作的干货？读者读完能学到什么？ |
| 🥈 中 | **持续性** | 不是发一篇就结束——持续输出才算 |
| 🥈 中 | **被传播** | 是否被别人主动转发、引用、讨论 |
| 🥉 加分 | **跨平台** | 同一内容适配多平台发布 |
| 🥉 加分 | **与其他挑战联动** | 文章内容来自 C1–C6 的实践 |
| 🥉 加分 | **内容生产 workflow 可复用** | 你的写作流程本身可以教给别人（C4 联动） |

**关于阅读量的现实预期：**

新号的第一篇文章，100 阅读就已经不错了。不要和大V比数据。我们看的是：

- **趋势**：第二篇比第一篇高？第五篇比第二篇高？→ 进步者
- **群内影响**：群友看了觉得有用？有人评论？→ 分享者
- **质量密度**：一篇有深度的 200 阅读 > 一篇水文的 2000 阅读

## 十、从零开始的最小路径

### 注册公众号（5 分钟）

```
1. 微信搜索"公众号"小程序
2. 注册个人订阅号（不需要企业资质）
3. 设置头像、名称、简介
4. 完成
```

### 发布第一篇文章（2 小时，AI 辅助）

```
Step 1：选一个你已经做过的事
        → C1 的翻译经验？C4 的技能创建过程？C6 的 App 踩坑？

Step 2：对 AI 说
        → "我做了____，过程是____，结果是____。
           帮我写一篇 1500 字的公众号文章，
           目标读者是对 AI 感兴趣的大学生，
           风格轻松实用，多用第一人称。"

Step 3：审校 AI 初稿
        → 加入你的真实细节（具体数字、具体工具名、具体踩坑经历）
        → 删掉 AI 生成的空洞套话
        → 检查事实准确性

Step 4：让 AI 生成 10 个标题候选
        → 选一个最吸引你自己点击的

Step 5：排版
        → 用公众号编辑器直接排版
        → 或让 AI 生成 HTML 后粘贴

Step 6：发布 + 分享到群里

Step 7：24 小时后截图数据，提交 C7
```
