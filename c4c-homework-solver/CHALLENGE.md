66|## 背景
67|
68|每个理工科学生都经历过这个痛苦循环：
69|
70|```
71|收到作业 PDF/Word → 手抄题目 → 手算/编程求解 → 手写 LaTeX → 编译 → 发现排版错误 → 修改 → 再编译 → 提交
72|```
73|
74|这个流程里，**真正有创造力的部分只有"求解"——其余全是搬运和格式工作。**
75|
76|如果有一个技能能：
77|
78|1. **自动读取**作业文件（PDF/Word/图片）→ 提取题目
79|2. **自动求解**数学/工程问题（符号计算 + AI 推理）
80|3. **自动排版**成专业的 LaTeX 文档
81|4. **自动编译**成 PDF 并验证
82|5. **一键提交**
83|
84|——那你每次做作业的时间就从 3 小时变成 15 分钟（审阅 + 修正）。
85|
86|**C4C 就是要你构建这条流水线——在国产大模型上。**

104|## 完整流水线架构
105|
106|```
107|┌─────────────────────────────────────────────────────────────┐
108|│  INPUT: 作业文件（PDF / Word / 图片 / LaTeX / Markdown）      │
109|└─────────────┬───────────────────────────────────────────────┘
110|              │
111|              ▼
112|┌─────────────────────────────────────────────────────────────┐
113|│  STAGE 1: 文档摄入 (Document Ingestion)                      │
114|│                                                              │
115|│  PDF → pdftotext / pdfplumber (文本型)                       │
116|│  PDF → Tesseract OCR (扫描型)                                │
117|│  Word → python-docx                                         │
118|│  Image → Tesseract OCR / 国产模型 Vision                     │
119|│  LaTeX → regex extraction                                   │
120|│  Markdown → 直接读取                                         │
121|│                                                              │
122|│  输出: 结构化 Markdown，含题号、题目文本、公式                 │
123|└─────────────┬───────────────────────────────────────────────┘
124|              │
125|              ▼
126|┌─────────────────────────────────────────────────────────────┐
127|│  STAGE 2: 题目解析 (Problem Parser)                          │
128|│                                                              │
129|│  识别题号（1, 2, 3... 或 (a), (b)...）                       │
130|│  提取数学表达式（LaTeX 公式、文字描述）                       │
131|│  分类题目类型（计算题、证明题、画图题、编程题）               │
132|│  识别子题（a, b, c）                                         │
133|│                                                              │
134|│  输出: List[Problem] — 每题含 id, text, math_expr, type      │
135|└─────────────┬───────────────────────────────────────────────┘
136|              │
137|              ▼
138|┌─────────────────────────────────────────────────────────────┐
139|│  STAGE 3: 自动求解 (Problem Solver)                          │
140|│                                                              │
141|│  SymPy Engine (符号计算):                                    │
142|│    - 方程求解 (solve, dsolve, linsolve)                      │
143|│    - 微积分 (diff, integrate, limit, series)                 │
144|│    - 线性代数 (Matrix, eigenvals, det, inv)                  │
145|│    - 微分方程 (dsolve, ode)                                  │
146|│                                                              │
147|│  国产 LLM Engine (推理增强):                                  │
148|│    - Qwen 3.6 / Kimi 2.5 替代 Claude 做推理                 │
149|│    - 证明题 / 文字题 / 概念题                                │
150|│    - SymPy 无法处理的复杂推理                                │
151|│    - 解题步骤的自然语言解释                                   │
152|│                                                              │
153|│  输出: List[Solution] — 每题含 steps[], answer, latex_expr   │
154|└─────────────┬───────────────────────────────────────────────┘
155|              │
156|              ▼
157|┌─────────────────────────────────────────────────────────────┐
158|│  STAGE 4: LaTeX 生成 (Document Renderer)                     │
159|│                                                              │
160|│  套用作业模板（课程名、学生姓名、日期、题号）                │
161|│  题目 + 解答交替排版                                         │
162|│  数学公式正确嵌入（align, equation, cases）                  │
163|│  图表插入（如有 matplotlib 绘图）                            │
164|│                                                              │
165|│  输出: homework.tex                                          │
166|└─────────────┬───────────────────────────────────────────────┘
167|              │
168|              ▼
169|┌─────────────────────────────────────────────────────────────┐
170|│  STAGE 5: 编译与验证 (Compile & Validate)                    │
171|│                                                              │
172|│  pdflatex / xelatex 编译（3 遍）                             │
173|│  检查编译错误 / 警告                                         │
174|│  验证 PDF 页数、文件大小、可读性                             │
175|│  （可选）对比原题验证答案正确性                              │
176|│                                                              │
177|│  输出: homework.pdf — 可直接提交                             │
178|└─────────────────────────────────────────────────────────────┘
179|```
