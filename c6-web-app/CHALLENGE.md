# Challenge 6：Web Application

> **目标：构建真实可运行的产品，不是 demo。**
> 你的 Web App 不是课堂作业——它是一个别人打开链接就能用的东西。

---

## 一、挑战目标

构建一个**完整的 Web Application**，必须集成 AI 能力，必须部署上线，任何人点开链接就能使用。

C6 是 Elite20 "生产能力"阶段的终点——从技能（C4）到开源项目（C5）到**可用产品**（C6）：

```
C4 你有一个能力
  → C5 你把它变成别人能协作的项目
    → C6 你把它变成别人打开浏览器就能用的产品
```

这一步的跃迁是：从"能用"到"好用"，从"我能跑起来"到"任何人都能跑起来"。

---

## 二、为什么是 Web App

| 能力 | Web App 怎么训练 |
|------|-----------------|
| **产品思维** | 你必须站在用户角度思考：打开页面第一眼看到什么？下一步做什么？ |
| **端到端交付** | 从代码到部署到可访问——完整的工程闭环 |
| **AI 集成能力** | 不是"用 AI 写代码"，而是"让 AI 成为产品的一部分" |
| **真实反馈** | 发到群里，别人用了就知道好不好——没有比这更真实的评审 |

即使你从来没写过前端代码，2026 年的 AI 工具完全可以帮你从零构建一个可用的 Web App。**关键不是你的编程水平，而是你的产品设计能力和 AI 驾驭能力。**

---

## 三、必须满足的要求

### 3.1 硬性要求

| 要求 | 说明 |
|------|------|
| **必须部署上线** | 有一个任何人都能访问的 URL，不是 localhost |
| **必须集成 AI 能力** | AI 是产品功能的核心部分，不是装饰 |
| **前端必须** | 用户能看到、能交互的界面 |
| **后端可选** | 纯前端 + API 调用也可以；有后端加分 |
| **必须可用** | 不是 demo，不是截图，不是"本地能跑"——是别人打开就能用 |

### 3.2 "可用"的定义

你的 App 至少要通过以下测试：

```
□ 一个从没见过这个 App 的人，打开链接后 30 秒内知道这是干什么的
□ 不需要注册/登录就能体验核心功能（或有 demo 模式）
□ 核心流程能走通：输入 → 处理 → 输出，没有中途报错
□ 在手机浏览器上也能基本使用（响应式布局）
□ 加载时间 < 5 秒
```

达不到这五条的不算"可用"，算"demo"。Demo 也可以提交，但评分会低于可用产品。

## 四、AI 集成的三个层次

C6 要求 AI 是产品的**核心能力**，不是点缀。以下是三个层次：

| 层次 | 描述 | 示例 | 评分 |
|------|------|------|------|
| 🟢 基础 | AI 做单一任务，用户触发 | 输入文本 → AI 翻译 → 显示结果 | 及格 |
| 🔵 进阶 | AI 嵌入多步骤 workflow | 上传文档 → AI 提取关键信息 → 生成报告 → 可下载 | 良好 |
| 🟣 深度 | AI 是持续交互的核心 | 对话式助手 / 自适应学习 / 实时分析 + 推荐 | 优秀 |

**不算 AI 集成的情况：**
- ❌ 用 AI 写了代码，但产品本身不调用 AI → 这是 AI 辅助开发，不是 AI 集成产品
- ❌ 页面上有一个"AI 生成"按钮但实际是预设文本 → 这是假集成

| 类型 | 示例 | 复杂度 |
|------|------|--------|

## 六、技术栈推荐

你不需要学会所有技术——选一条路线，让 AI 帮你搞定剩下的。

### 路线 A：最快上手（推荐新手）

```
前端：HTML + Tailwind CSS + vanilla JS
AI：直接调用 Claude / OpenAI API（前端 fetch）
部署：Vercel / Netlify（免费，GitHub 一键部署）
耗时：AI 辅助下 2-4 小时
```

### 路线 B：React 单页应用

```
前端：React + Tailwind + shadcn/ui
AI：Claude API / OpenAI API
部署：Vercel
耗时：AI 辅助下 4-8 小时
```

### 路线 C：全栈应用

```
前端：React / Next.js
后端：Python FastAPI / Node Express
AI：Claude API + 工具调用
数据库：SQLite / Supabase（免费层）
部署：Vercel（前端）+ Railway / Render（后端）
耗时：AI 辅助下 1-3 天
```

### 路线 D：零代码 / 低代码

```
工具：Claude Artifacts / v0.dev / Bolt / Lovable / Replit
AI：内置
部署：平台自带
耗时：1-2 小时
```

**选择标准：** 不追求复杂度，追求"可用 + 好用"。路线 A 做出一个好用的工具，比路线 C 做出一个半成品要强得多。

## 七、部署指南

部署是 C6 的关键门槛——本地能跑不算数。以下是三个免费部署方案：

### 方案 1：Vercel（推荐）

```bash
# 安装 Vercel CLI
npm i -g vercel

# 在项目目录下
vercel

# 按提示操作，2 分钟内拿到一个 .vercel.app 链接
```

也可以直接在 vercel.com 连接 GitHub repo，每次 push 自动部署。

### 方案 2：Netlify

类似 Vercel，适合纯静态站点。拖拽 HTML 文件到 netlify.com 即可。

### 方案 3：GitHub Pages

```bash
# 在 repo Settings → Pages → Source 选 main branch
# 几分钟后获得 https://你的用户名.github.io/项目名/
```

适合纯前端项目，不需要后端。

### 方案 4：Replit / Hugging Face Spaces

适合需要后端的 Python 项目。免费层有限制但足够 demo。

**API Key 安全提醒：** 绝对不要把 AI API Key 硬编码在前端代码里。使用环境变量（Vercel Environment Variables）或让用户自己输入 key。

## 八、两个场景化示例

### 示例 1：把 wechat-doc-mapper 变成 Web App

**用户体验：**

```
1. 用户打开网页
2. 看到一个文件上传区域 + 挑战说明
3. 用户拖拽多个文件进去（或选择文件夹）
4. 点击"分析"
5. 页面显示：
   - 按作者×挑战分组的表格
   - 缺口分析（哪些挑战还缺什么）
   - "下载 Excel"按钮
6. 整个过程 < 10 秒
```

**技术实现：**

```
前端：React + Tailwind
核心逻辑：JavaScript 版的文件解析（文件名正则 + 类型判断）
AI 集成：用 Claude API 对无法自动分类的文件做内容分析
部署：Vercel
```

**AI 集成点：** 当文件名不符合命名规范时，调用 Claude 读取文件前 500 字，推断作者和挑战归属。这就是 🔵 进阶层次的 AI 集成。

### 示例 2：把 skill-explainer 变成 Web App

**用户体验：**

```
1. 用户打开网页
2. 上传一个 .skill 文件（或粘贴 SKILL.md 内容）
3. 点击"X-Ray 分析"
4. 页面逐步展示 7 节分析报告：
   - 身份信息（带动画展开）
   - 用途说明
   - 架构图（自动生成目录树可视化）
   - 参考文件列表
   - 使用指南
   - 四维评审（D1–D4 评分卡，带颜色）
   - 改进建议（带优先级标签）
5. "导出 Markdown"按钮
```

**AI 集成点：** 整个分析过程由 Claude API 驱动——解析 SKILL.md、评估质量、生成建议。这就是 🟣 深度层次的 AI 集成。

## 九、AI-First 在 C6 的应用

Web App 项目**必须**使用 AI 辅助开发。`AI_LOG.md` 需记录：

| 阶段 | AI 怎么用 |
|------|----------|
| **产品设计** | 用 AI 生成产品需求文档、用户流程、UI 线框图 |
| **前端开发** | 用 AI 生成 React 组件、CSS 样式、页面布局 |
| **后端开发** | 用 AI 生成 API 路由、数据库 schema、中间件 |
| **AI 集成** | 用 AI 帮你写调用 AI 的代码（是的，meta！） |
| **调试** | 把报错信息丢给 AI，让它帮你 fix |
| **部署** | 用 AI 生成部署配置、环境变量设置、CI/CD |

**段位参考：**

| 段位 | 在 C6 的表现 |
|------|-------------|
| 🟢 入门 | 用 AI 生成单个页面，手动拼接 |
| 🔵 进阶 | 用 AI 生成完整项目骨架，迭代修改到可用 |
| 🟣 驾驭 | 设计完整 workflow，让 AI 分模块生成，自己做架构决策和集成 |
| 🔴 创造 | 用 Claude Code / Cursor 等工具端到端 vibe coding，产品本身也深度集成 AI |

## 十、拿来主义在 C6 的应用

Web App 是拿来主义的最佳实战场：

| 拿来什么 | 从哪拿 | 怎么改 |
|----------|--------|--------|
| UI 组件 | shadcn/ui、Tailwind UI、Chakra UI | 改颜色、布局、交互逻辑 |
| 页面模板 | v0.dev、Vercel 模板、开源 starter | 替换内容、调整结构 |
| API 架构 | FastAPI 模板、Express boilerplate | 添加自己的路由和业务逻辑 |
| AI 调用模式 | Anthropic Cookbook、OpenAI 示例 | 适配自己的 use case |
| 产品设计 | 参考已有产品的交互模式 | 改场景、改用户群、改功能组合 |

**ATTRIBUTION.md 必须写清楚：** 用了什么模板、什么组件库、参考了哪个产品的设计。

**特别鼓励：** fork 别人的 C5 repo → 加上前端 UI → 部署为 C6 Web App → 群内分享。这对你是 C6 贡献，对被 fork 的人是 C4/C5 的使用次数加分。

## 十一、评判标准

| 权重 | 指标 | 怎么衡量 |
|------|------|----------|
| 🥇 最高 | **可用性** | 打开链接就能用？核心流程走得通？不报错？ |
| 🥇 最高 | **AI 集成深度** | AI 是核心功能？还是点缀？达到哪个层次（🟢🔵🟣）？ |
| 🥈 中 | **用户体验** | 30 秒内知道这是干什么的？操作直觉？响应速度？ |
| 🥈 中 | **实用价值** | 解决了一个真实问题？别人会想再次使用？ |
| 🥉 加分 | **技术质量** | 代码结构清晰？有错误处理？有 loading 状态？ |
| 🥉 加分 | **移动端适配** | 手机上也能正常使用？ |
| 🥉 加分 | **持续迭代** | 根据群友反馈改进了 v2、v3？ |

**最关键的一条：** 把链接发到群里，观察有多少人试用、停留多久、是否回来再用。用户行为是最诚实的评审。

## 十三、从零开始的最小路径

**你完全不会写代码？没关系。以下路径全程 AI 驱动：**

```
Step 1：想清楚你要做什么
        → 一句话：输入____，输出____
        → 用户打开页面第一眼看到什么？

Step 2：让 AI 帮你生成完整代码
        → 对 Claude 说："帮我创建一个单页 Web App：
          用户上传 X，AI 分析后返回 Y，
          用 HTML + Tailwind + JavaScript，
          包含文件上传区域和结果展示区域。"
        → 你会拿到一个完整的 HTML 文件

Step 3：本地预览
        → 双击 HTML 文件在浏览器中打开
        → 看效果，不满意就让 AI 改

Step 4：添加 AI 集成
        → 对 Claude 说："在这个页面里加一个功能，
          用户点击'分析'按钮后调用 Claude API，
          把上传的文件内容发给 AI 分析，
          结果显示在页面上。"

Step 5：部署
        → 把文件推到 GitHub（见 C5 流程）
        → 在 vercel.com 连接 repo → 自动部署
        → 拿到 URL

Step 6：发到群里
        → 附一句话说明 + 邀请试用
```

**预计耗时：AI 辅助下 2–4 小时，从零到上线。**

如果你用 Claude Artifacts，甚至可以在对话中直接生成可交互的 React 组件，复制代码到 GitHub 即可部署。
