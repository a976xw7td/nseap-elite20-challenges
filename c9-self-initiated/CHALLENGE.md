# Challenge 9：Self-Initiated Project（自驱项目）

> **C8 证明你能"完成别人给的任务"，C9 证明你能"定义并完成自己的任务"。**
> 如果没有人给你方向、没有人给你 deadline、没有人监督你——你还能不能持续创造价值？

---

## 一、挑战定义

C9 要求你在 C8 之外，**并行启动一个完全由你驱动的项目**。

没有导师指定方向。没有客户定义需求。没有团队分配任务。只有你自己。

可以是创业想法，可以是个人兴趣，可以是自我提升系统，可以是你认为"应该存在但还不存在"的东西。唯一的要求：**它必须是真实的，你必须真的去做。**

---

## 二、C8 与 C9 的对偶关系

C8 和 C9 **必须同时进行**，不是先后关系。它们训练的是完全不同的能力：

| 维度 | C8 真实项目 | C9 自驱项目 |
|------|-----------|-----------|
| 驱动来源 | 外部（导师指定项目） | 内部（你自己定义） |
| 目标 | 交付结果给客户/组织 | 构建长期个人价值 |
| 约束 | 强（有需求、有 deadline） | 弱（你自己定规则） |
| 风险 | 低（方向明确） | 高（自己探索、可能走弯路） |
| 本质 | 执行能力 | 创造能力 |
| 失败模式 | 做不出来 | 不开始 / 不坚持 |

**为什么要同时做两个？** 因为真实世界需要两种人：能把别人的想法执行到位的人（C8），和能从零定义问题并解决的人（C9）。最稀缺的人两种都能做。

---

## 三、项目方向

不限方向，但必须是真实的。选择以下三类之一（或组合）：

### 🚀 类型 A：Startup Idea（创业方向）

你有一个商业想法，想验证它是否可行。

| 做什么 | 示例 |
|--------|------|
| AI 产品 / Agent 工具 | 面向特定行业的 AI 助手、自动化工作流工具 |
| 平台 / SaaS | 技能交易市场、AI 课程平台、垂直领域知识库 |
| 商业模式验证 | Landing page + 用户注册 → 验证有没有人要 |
| MVP | 最小可用产品 → 给 10 个人用 → 收集反馈 |

**C9 不要求你赚到钱——但要求你认真走完"发现需求→构建方案→验证假设"的循环。**

### 🎨 类型 B：Personal Interest（兴趣驱动）

你有一个真正感兴趣的领域，想深入下去。

| 做什么 | 示例 |
|--------|------|
| 内容品牌 | 围绕某个主题持续创作（与 C7 深度联动） |
| 开源工具 | 解决你自己遇到的问题，开源给别人用 |
| 研究探索 | 深挖一个技术/学术方向，产出论文或技术报告 |
| 社区 | 建立一个小型学习/兴趣社区 |

**判断标准：如果没有人要求你做，你还是会做——说明这是真的兴趣。**

### 🧠 类型 C：Life Improvement（人生系统）

把你自己变成一个更强的系统。

| 做什么 | 示例 |
|--------|------|
| 第二大脑 | 构建你的个人知识管理系统 + AI agent |
| 效率系统 | 自动化你的日常流程：邮件、日程、笔记、复盘 |
| 学习系统 | 设计你自己的学习路径 + 进度追踪 + 间隔重复 |
| 决策系统 | 用数据和 AI 辅助人生决策（职业、财务、健康） |

**这一类看起来最"软"，但做好了最有长期价值。** 一个运行良好的个人系统，是你所有其他能力的放大器。

### Proposal 五个必须部分

---

#### Part 1：项目定义（Project Definition）

```markdown
## 1. 项目定义

- 我想做什么：____（一句话）
- 输入：____
- 输出：____
- 为什么这件事重要：
  - 对我：____
  - 对他人 / 对世界：____
- 这个东西目前存在吗：
  - 如果存在：我的版本有什么不同？
  - 如果不存在：为什么之前没人做？我凭什么能做成？
```

**"为什么之前没人做"这个问题必须认真回答。** 答案通常是以下几种：

- 有人做过但做得不好 → 你要解释为什么你能做得更好
- 技术条件刚刚成熟（比如 AI 能力最近才到位）→ 合理
- 市场太小 → 如果你不在乎商业价值，个人项目可以
- 其实有人做了，你没找到 → 说明你的调研不够，回去做拿来主义

---

#### Part 2：拿来主义分析（必须）

C9 依然**强制遵守拿来主义原则**。即使是"完全原创"的想法，你也必须展示你做了调研。

```markdown
## 2. 拿来主义分析

### 竞品 / 参考项目
| 名称 | 链接 | 它做了什么 | 它做得好的地方 | 它做得不好的地方 |
|------|------|-----------|--------------|----------------|
| | | | | |
| | | | | |
| | | | | |

### 我拿来的部分
- 从____拿了：____
- 为什么好用：____

### 我不要的部分
- ____做了____但我不需要，因为：____

### 我的差异化
- 我的版本和现有方案的核心差异：____
- 这个差异为什么有价值：____
```

#### Part 3：自驱动机制（Motivation System）

> **这是 C9 独有的、最关键的部分。**

C8 有导师监督、有客户压力、有团队依赖。C9 什么都没有。你唯一的动力来源是你自己。

你必须回答：

```markdown
## 3. 自驱动机制

### 为什么你会持续做这个项目？
- 兴趣 / 好奇心：____
- 现实收益（能力增长 / 影响力 / 收入）：____
- 身份认同（"我是一个做____的人"）：____

### 如果没人监督，你如何保证推进？
- 我的节奏：（每天 X 小时？每周一个 milestone？）
- 我的公开承诺：（在哪里公开进度？群内？公众号？GitHub？）
- 我的最低标准：（即使最忙的一周，我至少也会做____）
- 我的"失败信号"：（如果连续____周没有产出，说明我应该____）
```

**为什么要写"失败信号"？** 因为自驱项目最大的敌人不是失败——是**缓慢死亡**。你不会某一天突然放弃，而是越来越忙、越来越拖、直到忘了。提前定义"什么情况说明我已经放弃了"，是对自己的诚实。

**动力类型参考：**

| 动力层次 | 描述 | 持久度 |
|----------|------|--------|
| 外部奖励 | 有人夸 / 有流量 / 有钱赚 | ⭐ 低（奖励消失就停） |
| 内部兴趣 | 觉得好玩 / 好奇 / 享受过程 | ⭐⭐⭐ 高 |
| 身份驱动 | "我是一个创造者" / "这就是我" | ⭐⭐⭐⭐ 最高 |

最好的 C9 项目来自第二层或第三层动力。如果你唯一的动力是"完成挑战拿分"，这个项目大概率坚持不下去。

```markdown
  - Week 3-4：____
  - Week 5-6：____
  - Week 7-8：____

### Â（Action Plan）
- 每个里程碑的具体步骤
- 使用什么工具 / AI / 方法

### R̂（Expected Result）
- 8 周后我预期的最终产出：____
- 成功标准：____
- 如何验证：____
```

---

#### Part 5：最小可行产出（MVP）

> **必须定义第一周能做出来的东西。必须是"可展示的"，不是想法。**

```markdown
## 5. MVP（第一周产出）

- 我第一周会做出来的东西：____
- 形态：（代码 / 文档 / 网页 / 视频 / 原型）
- 别人看了能理解什么：____
- 展示方式：____
```

**MVP 不需要好看，不需要完整。** 它的唯一目的是证明你**已经从想法进入了行动**。

示例：
- Startup → 一个 Landing page + 一句话价值主张
- 兴趣项目 → 一个能跑的最小 demo
- 人生系统 → 一个你已经用了一周的笔记模板 + AI workflow

## 五、执行要求

Proposal 通过后，正式进入 C9 执行阶段。以下为强制要求：

### 5.1 每周推进（Mandatory Weekly Output）

每周必须有**新的、可展示的产出**：

| 产出类型 | 示例 |
|----------|------|
| 新功能 | 增加了一个模块 / 页面 / API |
| 新内容 | 发布了一篇文章 / 录了一个视频 |
| 新验证 | 找了 5 个用户测试 / 收集了反馈数据 |
| 新迭代 | 基于反馈重构了某个部分 |

**"在思考"不算产出。"在调研"不算产出。"在学习"不算产出。** 这些都是过程，不是结果。C9 每周必须有一个**别人能看到的东西**。

### 5.2 强制公开（Public by Default）

你的项目**必须**通过至少一种渠道公开：

| 渠道 | 适合什么 |
|------|---------|
| GitHub | 代码、文档、技术项目 |
| 个人网站 | 产品、作品集 |
| 公众号 / 知乎 | 内容项目、进度分享（与 C7 联动） |
| 视频 / 社交媒体 | Demo、教程、进度 vlog |
| Elite20 群 | 所有项目的最低公开渠道 |

**不公开 = 不算完成。** 公开有两个作用：①让别人能用/能看/能反馈；②给你自己施加社会压力，防止缓慢放弃。

### 5.3 强制 AAR

每周提交：`姓名_C9_AAR_W周次.md`

```markdown
## C9 Weekly AAR — Week N

### 做了什么
- 本周产出：____
- 公开在哪里：____

### 哪些有效
- 什么方法 / 工具 / 策略起作用了：____

### 哪些无效
- 什么方法失败了：____
- 为什么失败：____

### 动力检查
- 本周的动力水平（1-10）：____
- 动力变化原因：____
- 需要调整什么来维持动力：____

### 下周计划
- 基于本周 AAR 调整的目标：____
```

**注意"动力检查"——这是 C9 AAR 独有的。** C8 的 AAR 关注执行偏差（ΔR），C9 的 AAR 还要关注**动力偏差**。动力持续下降是项目死亡的领先指标，提前发现才能提前干预。

## 六、评价标准

| 维度 | 核心问题 | 权重 |
|------|----------|------|
| **自驱力** | 是否持续推进？没人催也在做？ | 🥇 最高 |
| **清晰度** | 问题定义是否清楚？知道自己在做什么？ | 🥇 最高 |
| **执行力** | 是否真的做出来了？每周有可展示的产出？ | 🥈 高 |
| **进化能力** | 是否持续优化？Week 8 比 Week 1 好多少？ | 🥈 高 |
| **独特性** | 是否体现个人特点？不是千篇一律的作业？ | 🥉 加分 |

**自驱力怎么衡量？** 看两个信号：
1. **节奏稳定性** — 每周都有产出，而不是前两周猛干后面消失
2. **自我纠偏** — 遇到问题不是停下来等人救，而是自己找方案、调整方向

## 七、淘汰机制

| 行为 | 为什么不行 |
|------|-----------|
| ❌ 连续两周没有产出 | 停滞 = 放弃。C9 的核心就是持续推进。 |
| ❌ 只有想法，没有产出 | "我在想一个很厉害的项目"→ 那就做出来。 |
| ❌ 不公开 | 没人看到 = 没有反馈 = 没有进化 = 慢慢死掉。 |
| ❌ 不做 AAR | 不反思 = 重复犯同样的错。 |
| ❌ 不做拿来主义 | 闭门造车 = 浪费时间在已解决的问题上。 |
| ❌ 动力归零不处理 | AAR 中动力持续下滑但不调整 = 等死。换方向也比不动强。 |

**关于换方向：** C9 允许中途换项目，但必须在 AAR 中写清楚为什么换、从旧项目学到了什么。"我发现这个方向不对"是有效的学习成果，不是失败。但不能无限换——最多换一次。

## 八、C9 项目进化路径

一个好的 C9 项目会自然进化：

```
Week 1-2：MVP（最小可用，证明你开始了）
    ↓
Week 3-4：V1（核心功能完整，可以给人用了）
    ↓
Week 5-6：V2（基于反馈迭代，解决了真实问题）
    ↓
Week 7-8：V3（形成可持续的系统，你会继续做下去）
```

**最好的 C9 结果不是"完成了一个项目"，而是"启动了一个你会持续做下去的事情"。** Elite20 结束后你还在做 = 最高评价。

## 九、与其他挑战的联动

C9 不是孤立的——它可以和所有其他挑战互相加强：

| 联动 | 怎么做 |
|------|--------|
| C4 | 你的 C9 项目中产生的工具/方法，提取成 Skill 发到群里 |
| C5 | C9 项目的代码放在 GitHub，成为你的 C5 |
| C6 | C9 如果是产品类项目，部署上线就是 C6 |
| C7 | 每周公开进度/复盘文章 = C7 内容素材 |
| C8 | C9 的某个模块可能对 C8 的真实项目有用 → 跨项目复用 |

**最理想的状态：一个 C9 项目同时覆盖 C4 + C5 + C6 + C7 + C9 五个挑战。** 这不是偷懒——这是系统思维。

## 十三、Elite20 全局视图

C9 的加入使 Elite20 形成了完整的三层结构：

```
┌─────────────────────────────────┐
│  Layer 3: CREATE（创造）         │
│  C9 自驱项目                     │
│  "定义并完成自己的任务"           │
├─────────────────────────────────┤
│  Layer 2: EXECUTE（执行）        │
│  C8 真实项目                     │
│  "完成别人给的任务"              │
├─────────────────────────────────┤
│  Layer 1: LEARN（学习）          │
│  C1–C7 能力验证                  │
│  "证明你会"                     │
└─────────────────────────────────┘
```

这就是一个完整的**人才操作系统**：

- **Layer 1** 验证你有零件（技能、工具、表达能力）
- **Layer 2** 验证你能组装（在真实约束下交付结果）
- **Layer 3** 验证你能创造（从零定义问题并持续推进）

三层都完成的人，不再是"有潜力的学生"——而是**已经被验证的创造者**。

>
> C8 回答的是：你能不能执行？
> C9 回答的是：你能不能发起？
>
> 最稀缺的人不是能做事的人——而是知道该做什么事的人。
> C9 训练的就是这个。
