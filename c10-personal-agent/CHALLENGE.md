# Challenge 10：Build Your Own Agent（构建你的长期智能体 / 第二大脑）

> **C10 不是一个项目，而是你未来十年的"能力基础设施"。**
> 如果你离开 Elite20，你的能力是否还会持续增长？C10 就是让答案变成"是"的东西。

---

## 一、挑战定义

C1–C9 训练了你各种能力。但能力会遗忘、会退化、会过时。

C10 的目标是：**把你的所有能力沉淀成一个可持续进化的系统。**

你要构建的不是"一个 AI 应用"，而是：

| 它是什么 | 它做什么 |
|----------|---------|
| 你的知识系统 | 记住你学过的、做过的、失败过的一切 |
| 你的执行系统 | 把重复性任务变成可自动调用的 Skill |
| 你的决策系统 | 用积累的经验辅助你做判断 |
| 你的进化系统 | 每次使用都变得更强，而不是原地踏步 |

一个人 + 一个运行良好的 Agent 系统 = 一个**不会遗忘、持续进化、能力可累积**的超级个体。

## 二、C8 / C9 / C10 的三角关系

| Challenge | 本质 | 方向 |
|-----------|------|------|
| C8 | 在世界中执行 | 外 → 内（接受任务，交付结果） |
| C9 | 在自我中创造 | 内 → 外（定义问题，持续推进） |
| C10 | 构建"执行 + 创造"的系统 | 内 ↔ 内（把能力变成基础设施） |

C8 和 C9 是你做的事情。C10 是**做事情的系统本身**。

类比：C8 是你打的比赛，C9 是你自己组织的比赛，C10 是你的训练系统——让你在每场比赛后都变得更强。

## 三、系统五层架构（必须包含）

你的 Agent 系统必须包含以下五层。不需要每一层都很复杂，但每一层都必须存在且可工作。

```
┌──────────────────────────┐
│  Layer 5: Interface       │  你怎么和系统交互
├──────────────────────────┤
│  Layer 4: Tool / Access   │  系统能调用什么工具
├──────────────────────────┤
│  Layer 3: Agent Loop      │  KSTAR 认知循环
├──────────────────────────┤
│  Layer 2: Skill System    │  可复用的行动计划
├──────────────────────────┤
│  Layer 1: Memory          │  KSTAR 四层记忆
└──────────────────────────┘
```

### Layer 1：Memory（记忆系统）— KSTAR 结构

记忆是整个 Agent 的核心。没有记忆的 Agent 每次都从零开始——和没有 Agent 一样。

你的记忆系统必须实现**四层**：

| 层次 | 存什么 | 示例 | 存储形式 |
|------|--------|------|----------|
| **L1 原始输入** | 笔记、对话、信息、灵感 | 今天和客户开了一个会，讨论了 X | 文本 / Markdown |
| **L2 KSTAR 经验** | 结构化的 S→T→A→R 经验记录 | Situation: 客户要求 X → Task: 做 Y → Action: 用 Z 方法 → Result: 成功/失败 | KSTAR 格式 |
| **L3 Skill** | 从多次经验中抽象出的可复用能力 | "客户需求分析"技能：输入需求文档，输出结构化需求清单 | skill.md 格式 |
| **L4 Identity** | 工具能力、角色定义、价值观 | 我能调用 Claude API；我是 builder 角色；我的原则是…… | 配置文件 |

**L1 → L2 → L3 的转化是关键：**

```
L1 你记了一堆笔记（原始信息）
  ↓ 结构化
L2 你把笔记整理成 KSTAR 经验（从信息到知识）
  ↓ 抽象
L3 你从多条经验中提炼出 Skill（从知识到能力）
  ↓ 执行
L4 你的 Skill 被 Agent 调用去完成任务（从能力到行动）
```

**实现建议：**

| 方案 | 技术栈 | 适合谁 |
|------|--------|--------|
| 最简 | Markdown 文件 + 文件夹结构 | 不会编程的人 |
| 进阶 | SQLite 数据库（KSTAR_DB） | 会基础编程 |
| 高级 | 三存储模型（关系 + 属性图 + 向量） | 技术背景强 |

即使用最简方案（纯 Markdown），只要四层结构清晰，就是一个有效的记忆系统。

### Layer 2：Skill System（技能系统）

你必须把重复做的事情抽象成 Skill。每个 Skill 用以下格式表达：

```markdown
---
name: skill-name
description: 一句话说明 + 触发条件
---

# Skill 名称

## What（做什么）
一句话描述这个 Skill 的功能。

## When to Use（什么时候用）
触发条件和适用场景。

## Input（输入）
需要什么信息才能执行。

## Steps（步骤）
1. ____
2. ____
3. ____

## Tools（需要什么工具）
- LLM / API / 脚本 / ……

## Output（输出）
执行完成后产出什么。
```

**Skill = 可复用的行动计划（Action Plan）。** 不是文档，不是笔记——是可以被你的 Agent 调用执行的东西。

**技能生命周期：**

```
经验（L2）→ 候选技能 → 验证 → 上线 → 优化 → 可能淘汰
```

- 从 2-3 条类似经验中提炼出候选 Skill
- 手动执行一次验证有效
- 纳入 Skill 库，标记为"可用"
- 使用过程中持续优化
- 如果某个 Skill 连续 N 次不被调用或效果差 → 淘汰或重构

### Layer 3：Agent Loop（KSTAR 认知循环）

这是你的系统"活起来"的核心。每次使用都走一遍完整循环：

```
K（Knowledge）: 从 Memory 中检索相关知识和 Skill
    ↓
S（Situation）: 评估当前情境（我面对什么？有什么约束？）
    ↓
T（Task）: 明确任务（我要做什么？分几步？）
    ↓
Â（Action Plan）: 生成行动计划 + 预测预期结果 R̂
    ↓
A（Action）: 执行计划（调用工具、运行 Skill）
    ↓
R（Result）: 记录实际结果
    ↓
ΔR = R̂ − R: 计算偏差（预期 vs 实际）
    ↓
更新 Memory（新经验写入 L2，可能触发 L3 Skill 更新）
```

**ΔR 是学习信号：**
- ΔR ≈ 0 → 你的预测准确，Skill 有效，继续用
- ΔR 大 → 需要校准，可能要修改 Skill 或更新知识
- 连续多次 ΔR 大 → Skill 可能过时，需要重构

**最简实现：** 不需要写代码。每次做事前写下 S→T→Â→R̂，做完后记录 R 和 ΔR。用 Markdown 就够了。关键是**养成循环习惯**，不是技术实现。

### Layer 4：Tool / Access System（工具层）

你的 Agent 能调什么工具决定了它的"行动半径"。

对应公式：**Action = Plan × Access Token × Energy**

- **Plan** = Skill 告诉你做什么
- **Access Token** = 你能调用哪些工具/API/服务
- **Energy** = 你（或 AI）实际投入的执行力

你必须明确列出你的工具清单：

| 类别 | 工具 | 用途 |
|------|------|------|
| LLM | Claude / GPT / DeepSeek / 本地模型 | 推理、生成、分析 |
| 代码 | Claude Code / Cursor / VS Code | 编程、自动化 |
| 知识管理 | Obsidian / Notion / Markdown 文件 | 记忆系统载体 |
| 版本控制 | GitHub | Skill 存储、项目协作 |
| 浏览器 / API | 搜索、数据获取、外部服务 | 信息输入 |
| 部署 | Vercel / Railway | 产品上线 |
| 通讯 | 微信 / 邮件 | 社会交互 |

**不需要工具很多。** 3-5 个核心工具 + 清晰的调用方式 > 20 个工具但不知道怎么串联。

### Layer 5：Interface（交互层）

你怎么和你的 Agent 系统交互？至少实现一种：

| 方式 | 适合谁 | 实现难度 |
|------|--------|----------|
| **Markdown 驱动** | 所有人 | 🟢 最简（文件夹 + 模板） |
| **CLI** | 会命令行的人 | 🔵 推荐（Claude Code / 脚本） |
| **Chat UI** | 想做产品的人 | 🟣 进阶（Web 界面） |
| **Voice** | 实验性 | 🔴 高级（语音输入 + TTS 输出） |

**Markdown 驱动系统示例：**

```
my-agent/
├── memory/
│   ├── L1_raw/           （日常笔记和输入）
│   ├── L2_episodes/      （KSTAR 经验记录）
│   └── L3_skills/        （提炼的 Skill 文件）
├── config/
│   ├── identity.md       （我是谁、我的原则）
│   └── tools.md          （我能用什么工具）
├── inbox/                （待处理的输入）
├── logs/                 （Agent 执行日志）
└── AGENT.md              （Agent 主配置：循环逻辑、调用规则）
```

即使完全没有代码，这个文件夹结构 + Claude 对话就构成了一个可用的 Agent 系统。每次和 Claude 对话时，把 `AGENT.md` + 相关 Skill + 相关记忆喂给它，就是一次 Agent Loop 执行。

## 四、必须体现的三个原则

### ✅ 原则 1：拿来主义

你的系统不需要从零设计。必须写清楚：

```markdown
## 拿来说明

### 参考系统
| 系统 | 我拿了什么 | 我改了什么 |
|------|-----------|-----------|
| Obsidian | 文件夹结构 + 双链笔记理念 | 用 KSTAR 替代自由标签 |
| Claude Code Skill 体系 | skill.md 格式 + 三层加载 | 加了生命周期管理 |
| PARA 方法 | 四层分类思想 | 用 KSTAR 四层替代 PARA 四层 |
| LangChain | Agent + Tool 调用模式 | 简化为 Markdown 驱动 |
```

### ✅ 原则 2：自进化（Self-Evolving）

你的系统必须能够：

| 能力 | 怎么做 |
|------|--------|
| **从经验中生成 Skill** | 当你发现自己连续 3 次用类似方法做事 → 抽象成 Skill |
| **优化已有 Skill** | 当 ΔR 持续偏大 → 修改 Skill 的步骤或工具 |
| **淘汰无效 Skill** | 当 Skill 连续 N 次不被调用或效果差 → 标记为 deprecated |
| **Skill 组合** | 当两个 Skill 经常一起调用 → 合并成一个更强的 macro-skill |

系统不自进化 = 你只是做了一个静态的文档库。Agent 和文档库的区别就在于：**Agent 会从使用中学习。**

### ✅ 原则 3：长期性（Persistent）

| 要求 | 含义 |
|------|------|
| 数据可积累 | 不是一次性使用——每次使用后数据留下来 |
| 系统可扩展 | 加新 Skill、新工具不需要重构整个系统 |
| 可持续使用 | Elite20 结束后你还在用 = 最高验证 |

**反例：** 如果你的"Agent 系统"只在提交 C10 的那一周用了一次，之后再也没打开过——它就不是一个系统，只是一次作业。

### 使用记录要求

不是"我设计了什么"，而是"我真的用了，效果如何"：

```markdown
## Usage Log

### 使用场景 1：用 research-scanner 调研竞品
- 日期：2026-03-15
- 输入：「AI 教育平台竞品分析」
- 调用的 Skill：research-scanner
- 实际效果：30 分钟找到 8 个竞品，比之前手动调研快 3 倍
- ΔR：预期找到 10 个，实际 8 个（2 个被搜索引擎过滤了）
- 改进：给 Skill 加了多搜索引擎并行的步骤

### 使用场景 2：用 weekly-planning 做周计划
- 日期：2026-03-17
- 输入：本周 OKR + 上周 AAR
- 调用的 Skill：weekly-planning
- 实际效果：从 45 分钟的周计划缩短到 15 分钟
- ΔR：计划完成率从上周 60% 提升到本周 80%
- 改进：加了"上周未完成任务自动继承"的步骤
```

## 十一、Elite20 全局闭环

C10 的完成使整个 Elite20 形成最终闭环：

```
┌─────────────────────────────────────────┐
│                                         │
│   C1–C3   输入能力（获取信息）            │
│     ↓                                   │
│   C4–C7   转化能力（技能、表达、产品）      │
│     ↓                                   │
│   C8      外部执行（真实世界交付）          │
│     ↓                                   │
│   C9      内部驱动（自我定义 + 创造）       │
│     ↓                                   │
│   C10     系统沉淀（长期进化基础设施）      │
│     ↓                                   │
│     └───→ 回到 C1（用更强的系统获取新知识） │
│                                         │
└─────────────────────────────────────────┘
```

**本质就是把一个人训练成一个 Agent System：**

```
Learn（C1–C7）→ Execute（C8）→ Create（C9）→ Systematize（C10）→ Loop
```

C10 不是终点——它是让这个循环**永远不停**的引擎。
