# Challenge C3C：双人协作与技能迁移（Peer Collaboration & Skill Transfer）

> 🔥 不是两个人各做一半——是两个人互相教会对方原本不会的东西。
> 本质：证明 **1 + 1 > 2**——通过协作，每个人都获得了独自无法获得的能力。

---

## 🤔 这是什么

AI 时代最稀缺的不是技术——是**跨领域协作能力**。一个懂代码的人和一个懂业务的人组队，能做出任何一方单独做不出的东西。

C3C 要求你：

1. **找一个搭档**组成二人小组
2. **至少一人必须是非计算机专业**
3. 完成一个**任何一方单独无法完成**的项目
4. 清晰展示**知识迁移**——谁教了谁什么、学之前和学之后有什么变化

> ⚠️ **不是分工——是互教。** "你写代码我写文档"不算。"你教我写代码、我教你做需求分析"才算。

---

## 🔑 核心规则

### 规则 1：必须跨专业

| 有效组合 ✅ | 无效组合 ❌ |
|-------------|-------------|
| 计算机 + 金融 | 计算机 + 软件工程 |
| 计算机 + 教育 | 计算机 + 计算机 |
| 计算机 + 设计 | 同专业两人 |
| 计算机 + 医学 | — |
| 任何非CS + 非CS（可以，只要有技能差异） | — |

### 规则 2：必须有知识迁移

每个人必须在项目中**学会一项自己之前不会的技能**，并有证据证明。

| 角色 | 教出去的 | 学到的 |
|------|----------|--------|
| CS 同学 | Python 编程 / Git 操作 / API 调用 | 商业分析 / 用户研究 / 学科知识 |
| 非 CS 同学 | 领域知识 / 需求设计 / 内容创作 | 编程基础 / 工具使用 / 技术实现 |

### 规则 3：单独做不到

项目成果必须是**任何一方单独无法完成的**。提交时需要明确说明：
- 如果没有搭档 A，搭档 B 缺少什么能力来完成这个项目？
- 如果没有搭档 B，搭档 A 缺少什么能力来完成这个项目？

---

## 🎯 四级任务

### Level 1：组队 + 技能画像（Bronze 🥉）

> **找到搭档，互相了解对方会什么、不会什么。**

- 找到一个跨专业搭档
- 各自完成一份**技能自画像**：
  - 我的专业和背景
  - 我已经会的技能（列举 5–10 项）
  - 我想学的技能（列举 3–5 项）
  - 我能教别人的技能（列举 3–5 项）
- 共同确定一个双方都有兴趣的项目方向
- 写一份**协作方案**：谁教谁什么、项目目标是什么、为什么单独做不了

**交付：** 两人技能自画像 + 协作方案文档

---

### Level 2：知识迁移 + 联合产出（Silver 🥈）

> **互相教学，并产出一个联合作品。**

- 完成至少 **2 次有记录的教学会话**（截图 / 录屏 / 聊天记录）
- 每次教学后，学习方写一份简短的**学习笔记**（学了什么、卡在哪里、怎么突破的）
- 联合完成一个项目产出，可以选择：

| 项目类型 | 示例 |
|----------|------|
| **技术 + 领域** | CS 同学教非 CS 同学用 Python 分析该同学专业的数据集 |
| **工具 + 内容** | CS 同学教 Git，非 CS 同学用 Git 管理自己的论文/作品 |
| **Agent + 应用** | 共同构建一个解决非 CS 同学专业问题的 Agent 技能 |
| **已有挑战联合完成** | 一起完成 C4C / C4D / C5A / C6A 中的一个 |

**交付：** 教学记录 + 学习笔记 + 联合产出物

---

### Level 3：双向迁移 + 能力验证（Gold 🥇）

> **不只是一方教另一方——双向都要有成长。**

在 Level 2 基础上：

- **双向知识迁移：** CS 同学也必须从非 CS 同学那里学到新知识/技能
- **能力验证：** 每人独立完成一个小任务，证明自己确实学会了新技能：
  - 非 CS 同学：独立写一段代码 / 独立跑通一个工具 / 独立完成一次 Git 提交
  - CS 同学：独立完成一个领域分析 / 独立产出一份专业内容 / 独立使用搭档教的方法论
- **前后对比：** 提供"学之前"和"学之后"的能力对比证据

**交付：** 双向教学记录 + 独立能力验证截图 + 前后对比报告

---

### Level 4：可复制的协作模式（Platinum 💎）

> **不只是你们两个成功了——把协作方法论提炼出来，让别人也能复制。**

在 Level 3 基础上：

- **协作方法论文档：** 提炼你们的协作模式——怎么找搭档、怎么设计教学、怎么互相给反馈、踩了什么坑
- **模板化：** 创建一个可复用的"跨专业协作模板"，让下一对搭档可以直接套用
- **公开分享：** 用 C4B 技能写成公众号文章，或在群里做一次分享
- **项目完整度：** 联合产出物达到可对外展示的质量

**交付：** 协作方法论文档 + 可复用模板 + 公开分享链接 + 高质量联合产出

## 💡 项目灵感

不知道做什么项目？这里有一些跨专业组合的灵感：

| 组合 | 项目灵感 | CS 教什么 | 非 CS 教什么 |
|------|----------|-----------|-------------|
| CS + 金融 | 用 Python 分析股票数据并可视化 | Python / pandas / 数据可视化 | 财务指标含义 / 投资分析逻辑 |
| CS + 教育 | 构建一个学科知识问答 Agent | API 调用 / prompt 工程 | 教学设计 / 知识点拆解 |
| CS + 设计 | 用 AI 生成品牌视觉方案并建网站 | HTML/CSS / 部署 | 设计原则 / 品牌策略 |
| CS + 医学/护理 | 构建一个健康数据追踪工具 | 数据库 / 前端 | 医学指标解读 / 用户需求 |
| CS + 外语 | 构建一个多语言学习 Agent | Agent 技能开发 | 语言学知识 / 教学方法 |
| CS + 体育 | 用数据分析运动表现 | 数据分析 / 可视化 | 运动科学 / 训练理论 |
| 非 CS + 非 CS | 任何有明确技能差异的组合都行 | — | — |

> 也可以联合完成一个已有挑战（C4C / C4D / C5A / C6A），只要满足知识迁移要求。

## 📎 需要提交什么（两人联合提交）

```
组名_C3C_协作方案.md               ← 谁和谁、各自背景、项目目标、为什么单独做不了
组名_C3C_技能画像_姓名A.md        ← 搭档 A 的技能自画像
组名_C3C_技能画像_姓名B.md        ← 搭档 B 的技能自画像
组名_C3C_教学记录/                 ← 教学会话截图/录屏/聊天记录（至少 2 次）
组名_C3C_学习笔记_姓名A.md        ← A 的学习笔记（学了什么/卡在哪/怎么突破）
组名_C3C_学习笔记_姓名B.md        ← B 的学习笔记
组名_C3C_联合产出/                 ← 项目成果（代码/文档/网站/技能包等）
组名_C3C_前后对比.md               ← 每人"学之前 vs 学之后"的能力对比
组名_C3C_AAR.md                    ← 双人联合复盘：做了什么 / 学了什么 / 怎么验证的
组名_C3C_AI日志.md（必须）         ← 双人 AI 使用全过程。无此项无法评审。
```

> **命名中的"组名"：** 用两人姓名拼音组合，如 `zhangsan-lisi_C3C_...`

---

## 评审标准

| 维度 | 权重 | 说明 |
|------|------|------|
| **知识迁移证据** | 30% | 是否有清晰的"教"和"学"的过程记录？双向的吗？ |
| **联合产出质量** | 25% | 项目成果是否完整、有价值、达到展示水准？ |
| **不可独立完成性** | 20% | 是否令人信服地说明了单独做不到？ |
| **前后对比** | 15% | 能力变化是否有具体证据（截图、代码、产出）？ |
| **AAR 与反思** | 10% | 复盘是否真实、有深度、有可复制的经验？ |

## ⚡ 起步指南

### Step 1：找搭档（Day 1）

在群里喊一声，或者私聊一个你觉得互补的同学。关键是**技能差异**——你们会的东西越不同，C3C 的价值越大。

### Step 2：互相了解 + 定项目（Day 1–2）

各写一份技能自画像，然后坐下来（或视频通话）聊：
- 你想学我什么？
- 我想学你什么？
- 我们能一起做个什么项目？

### Step 3：交替教学（Day 2–5）

安排至少 2 次教学会话。建议：
- 第 1 次：A 教 B（记录过程）
- 第 2 次：B 教 A（记录过程）
- 每次教学后各自写学习笔记

### Step 4：联合产出（Day 3–6）

一起完成项目。过程中持续记录谁做了什么、谁教了谁什么。

### Step 5：验证 + 复盘（Day 6–7）

各自独立完成一个小任务证明学会了新技能。然后坐下来一起写 AAR。

## 为什么 C3C 值得做

```
C3 系列 = 协作能力
C3A = 我会给别人反馈
C3B = 我会接受反馈并改进
C3C = 我会跨专业协作，互相教学，共同产出
```

**AI 时代的真相：** 最强的团队不是"所有人都会写代码"的团队，而是"每个人都能快速学会队友的核心技能"的团队。

C3C 训练的就是这个能力：

- **教学是最好的学习：** 你教别人的过程中，自己的理解会加深十倍
- **跨领域 = 创新：** 所有突破性产品都诞生在学科交叉点
- **协作记录 = 简历亮点：** "我教会了一个非技术背景的同学独立完成 Git 提交"——这比任何单人项目都更能证明你的沟通能力

---

> 📣 **C3C 的终极价值：**
>
> 单打独斗能走快。协作互教能走远。
> 不是你会什么决定你的上限——是你能**教会别人什么**、能**从别人那里学到什么**。
> 找一个和你完全不同的人，一起做一件你们各自做不到的事。
> 这就是 C3C。
